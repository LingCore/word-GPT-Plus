# Word GPT Plus 安装指南

本文档面向非技术人员，手把手教你将 Word GPT Plus 插件安装到 Microsoft Word 中。

---

## 前提条件

| 项目 | 要求 |
|------|------|
| Word 版本 | Word 2016/2019 零售版、Word 2021 或 Microsoft 365 |
| 文档格式 | 仅支持 `.docx`，不支持旧版 `.doc` |
| 运行时 | 需安装 [Edge WebView2 Runtime](https://developer.microsoft.com/en-us/microsoft-edge/webview2/)（Win10/11 通常已自带） |
| API 密钥 | 至少需要一个 AI 服务商的 API Key（OpenAI / Azure / Gemini / Groq / Ollama） |

---

## 第一步：获取插件服务

根据你的情况选择一种方式：

### 方式 A：使用公共服务（最简单）

直接下载 `manifest.xml`，无需搭建任何服务。

1. 下载文件：[manifest.xml（即刻使用版）](https://github.com/Kuingsmile/word-GPT-Plus/blob/master/release/instant-use/manifest.xml)
2. 右键 → 另存为，保存到电脑上一个专门的文件夹，例如：`C:\Users\你的用户名\Documents\WordGPT\`

> 如果遇到网络问题无法使用，请改用方式 B。

### 方式 B：使用内网自托管服务（推荐团队使用）

如果团队已部署了 Word GPT Plus 服务（Docker 或源码构建），按以下步骤操作：

1. 找管理员获取服务器地址（例如 `http://192.168.1.100:3000`）
2. 下载文件：[manifest.xml（自托管版）](https://github.com/Kuingsmile/word-GPT-Plus/blob/master/release/self-hosted/manifest.xml)
3. 用记事本打开 `manifest.xml`，将里面所有的 `http://localhost:3000` 替换为实际的服务器地址
4. 保存到电脑上一个专门的文件夹，例如：`C:\Users\你的用户名\Documents\WordGPT\`

---

## 第二步：共享文件夹

Word 插件要求 `manifest.xml` 所在的文件夹必须是 Windows 网络共享文件夹。

1. 打开**文件资源管理器**，找到你保存 `manifest.xml` 的文件夹（如 `C:\Users\你的用户名\Documents\WordGPT`）
2. **右键**该文件夹 → 选择**属性**
3. 点击**共享**选项卡 → 点击**共享(S)...** 按钮
4. 在弹出窗口的输入框中输入 `Everyone` → 点击**添加**
5. 将 Everyone 的权限设为**读取** → 点击**共享**
6. 系统会显示一个**网络路径**（类似 `\\电脑名\WordGPT`），**请记下这个路径**，后面要用
7. 点击**完成** → 点击**关闭**

---

## 第三步：在 Word 中添加受信任目录

1. 打开 **Word**，新建或打开任意 `.docx` 文档
2. 点击菜单栏 **文件** → **选项**
3. 在左侧列表选择 **信任中心** → 点击右侧的 **信任中心设置...** 按钮
4. 在左侧选择 **受信任的加载项目录**
5. 在 **目录 URL** 输入框中，粘贴第二步记下的**网络路径**（例如 `\\电脑名\WordGPT`）
6. 点击 **添加目录**
7. 勾选新添加条目右侧的 **显示在菜单中**
8. 点击 **确定** 关闭信任中心
9. 点击 **确定** 关闭 Word 选项
10. **完全关闭 Word**（确保任务栏也没有 Word）

---

## 第四步：加载插件

1. 重新打开 **Word**，新建或打开一个 `.docx` 文档
2. 点击菜单栏 **插入** → 点击 **我的加载项**（或 **获取加载项**）
3. 在弹出窗口顶部切换到 **共享文件夹** 选项卡
4. 选择 **Word GPT Plus**（或 **GPT Plus**）
5. 点击 **添加**
6. Word 右侧会弹出插件面板，安装完成

---

## 第五步：配置 API 密钥

1. 在右侧插件面板中，点击顶部的 **齿轮图标**（设置）
2. 切换到 **API Provider** 选项卡
3. 选择你要使用的 AI 服务商（OpenAI / Azure / Gemini / Groq / Ollama）
4. 填入对应的 API Key 和其他必要信息
5. 返回主页即可开始使用

### 各服务商 API Key 获取方式

| 服务商 | 获取地址 | 说明 |
|--------|----------|------|
| OpenAI | [platform.openai.com](https://platform.openai.com/account/api-keys) | 需注册并绑定付款方式 |
| Azure OpenAI | [Azure 门户](https://portal.azure.com/) | 需申请 Azure OpenAI 服务 |
| Google Gemini | [AI Studio](https://aistudio.google.com/apikey) | 免费额度可用 |
| Groq | [console.groq.com](https://console.groq.com/keys) | 免费额度可用 |
| Ollama | 无需 Key | 需在本机安装 [Ollama](https://ollama.com/)，仅限本地使用 |

## 常见问题

### Q：共享文件夹里显示"当前没有加载项可用"

- 确认文件夹已经按第二步完成了 Windows 网络共享（右键 → 属性 → 共享选项卡应显示"已共享"）
- 确认信任中心里填写的是**共享后的网络路径**（`\\电脑名\文件夹名`），而不是本地路径（`C:\...`）
- 确认勾选了"显示在菜单中"
- 确认已经完全关闭并重新打开了 Word

### Q：插件面板显示空白或加载失败

- 如果使用**方式 A（公共服务）**：检查网络是否能访问外网
- 如果使用**方式 B（自托管）**：确认服务器正在运行，在浏览器中打开服务器地址（如 `http://192.168.1.100:3000`）应能看到页面
- 检查是否安装了 [Edge WebView2 Runtime](https://developer.microsoft.com/en-us/microsoft-edge/webview2/)

### Q：发送消息后没有回复

- 检查是否已在设置中正确配置了 API Key
- 检查 API Key 是否有效（余额是否充足）
- 如果使用 Ollama，确认 Ollama 服务正在本机运行

### Q：我的 API Key 安全吗？

所有 API Key 和设置都存储在你本机的浏览器本地存储中，不会发送到任何第三方服务器。插件直接与你选择的 AI 服务商通信。

---

## 日常使用

安装完成后，以后每次打开 Word 文档，只需点击菜单栏的 **GPT Plus** 按钮即可打开插件面板（首次添加后会出现在工具栏中）。如果工具栏没有该按钮，重复第四步的操作即可。
